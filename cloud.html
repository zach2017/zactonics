01 — The Grocery Store Story

Imagine your company's data system is a big grocery store. Every department has a job. Every shelf has a purpose. Every receipt tells a story. If one part breaks, the whole store gets weird. ETL — Extract, Transform, Load — is just a fancy way of saying "get the stuff, clean it up, and put it on the right shelf."

In a real grocery store, raw produce arrives on a truck, gets washed and labeled, then placed where customers can find it. In our data store, raw files and events arrive from outside apps, get cleaned and tagged by NiFi, streamed by Kafka, and stored where searchers can find them — in OpenSearch, S3, and Postgres.

The whole guide uses this metaphor. Whenever you see 🛒 GROCERY, that's the simple-English version of what just happened.

🛒 GROCERY · Your AWS account is the grocery store building. Your VPC is the floor plan with aisles. Subnets are individual aisles. Security groups are the doors that decide who gets through.

02 — Services and Their Jobs

▎LOADING DOCK

Upload Form (S3 + Lambda)

A web form where users upload PDFs, Word docs, scans, etc. Lambda runs textract to pull words out, then sends them downstream.

🛒 The back door where delivery trucks drop off boxes of new product.
▎CONVEYOR BELT

Apache NiFi 2.9.0

The visual flow tool that grabs files, calls Textract for OCR, splits records, and routes each one to its destination. Drag-and-drop, no big code rewrites.

🛒 The stock clerks who unbox the truck, sort items by aisle, and label every box.
▎INTERCOM SYSTEM

Apache Kafka 4.2.0 (KRaft)

A streaming bus. Anything that needs to be shared across teams in real time gets published to a topic. Other services subscribe. Kafka 4.x runs in KRaft mode — no more ZooKeeper.

🛒 The store-wide intercom. "Lane 3 is open!" Everyone hears it instantly.
▎FILE CABINET

Amazon S3

Object storage. Every original document gets dropped here with versioning and lifecycle rules. Cheap, durable, infinite. Source of truth for raw docs.

🛒 The giant warehouse out back. Every box that's ever come in is filed there forever.
▎SEARCH KIOSK

Amazon OpenSearch 3.5

Full-text search and analytics. Every doc gets indexed so external apps can search words inside any uploaded file in milliseconds.

🛒 The "Where do I find olives?" kiosk near the front door. Type and find anything.
▎CASH REGISTER LEDGER

Amazon RDS PostgreSQL 16

Relational database for structured records — sales, inventory levels, HR rows, customer feedback. Joins, transactions, the works.

🛒 The accountant's ledger book where every transaction lines up neatly in columns.
▎AISLE MAP

VPC + Subnets

The private cloud network. Public subnets face the internet (load balancer, upload form). Private subnets hide the brokers and databases.

🛒 The store layout. Aisles. Walls. Doors marked "employees only."
▎SECURITY GUARDS

IAM + Security Groups + KMS

Permissions, firewalls, and encryption keys. Decides who can enter which aisle, what they can touch, and what's locked behind glass.

🛒 The badge readers, the cameras, and the manager's keyring.

  • → A registered domain (optional but recommended for HTTPS)

01 AWS Account Setup

Run the bootstrap script. It creates a Terraform-state S3 bucket, a DynamoDB lock table, and a least-privilege deploy role.

# From your laptop
./scripts/00-bootstrap-aws.sh us-east-1 my-company-etl

What the script does, in plain English:

  1. 1. Creates an encrypted S3 bucket to hold Terraform state (my-company-etl-tfstate)
  2. 2. Creates a DynamoDB table for state locking so two engineers can't apply at once
  3. 3. Creates a KMS key for encrypting everything-secret
  4. 4. Creates an IAM role etl-deployer that Terraform will assume
  5. 5. Prints the values you'll paste into terraform/backend.hcl
🛒 GROCERY · Before you stock the shelves you need a building, a safe to hold the money, and a master key. This step buys you all three.

02 Terraform VPC & Network

Build the VPC, subnets, NAT gateway, security groups, and VPC endpoints.

cd terraform/
terraform init -backend-config=backend.hcl
terraform plan  -target=module.network -out=network.tfplan
terraform apply network.tfplan

Files involved:

  • → terraform/network.tf — VPC, subnets, route tables, NAT
  • → terraform/security.tf — security groups, NACLs
  • → terraform/variables.tf — region, CIDR ranges, tags
🛒 GROCERY · You poured the foundation, framed the walls, put up "STAFF ONLY" signs on the back rooms, and ran a private hallway to the dumpster (NAT) so trash can leave without customers seeing it.

03 Terraform Managed Services

Stand up the managed AWS services that don't need EC2: S3 buckets, RDS Postgres, OpenSearch domain, Lambda, API Gateway.

terraform plan  -target=module.data -out=data.tfplan
terraform apply data.tfplan

Files involved:

  • → terraform/s3.tf — three buckets: raw, processed, archive (lifecycle to Glacier)
  • → terraform/rds.tf — PostgreSQL 16 Multi-AZ with KMS encryption
  • → terraform/opensearch.tf — OpenSearch 3.5 domain in VPC
  • → terraform/lambda.tf — Textract extractor function
  • → terraform/api_gateway.tf — pre-signed upload URL endpoint
🛒 GROCERY · The shelves arrive pre-built. AWS installs them, plugs them in, and you just say where to bolt them down.

04 Terraform Compute (EC2 for NiFi & Kafka)

Provision the EC2 instances that will run NiFi and Kafka. Terraform writes their IPs to an Ansible inventory file.

terraform plan  -target=module.compute -out=compute.tfplan
terraform apply compute.tfplan

# Inventory is auto-generated:
cat ../ansible/inventory/aws_ec2.yml

Defaults: 3 × m7i.xlarge for NiFi, 3 × m7i.large for Kafka brokers (combined mode KRaft controllers + brokers). All in private subnets, accessed only via SSM Session Manager.

🛒 GROCERY · You hired the stock clerks (NiFi) and the intercom operators (Kafka). They show up with empty toolbelts — next step is teaching them their jobs.

05 Ansible Inventory

Ansible reads the dynamic inventory and uses SSM to reach private hosts — no SSH keys to manage.

cd ansible/
ansible-inventory --list -i inventory/aws_ec2.yml | jq '.kafka_brokers.hosts, .nifi_nodes.hosts'
🛒 GROCERY · You wrote down everyone's locker number so you can find them later without yelling across the store.

06 Ansible Installs Kafka (KRaft)

Run the Kafka playbook. It installs Java 21, downloads Kafka 4.2.0, formats the KRaft metadata, and starts the three brokers as a quorum.

ansible-playbook -i inventory/aws_ec2.yml playbooks/kafka.yml

Important — there is no ZooKeeper anymore.

Apache Kafka 4.0 (March 2025) removed ZooKeeper support permanently. Kafka 4.2.0 (Feb 2026) uses KRaft mode exclusively — controllers and brokers live in the same Kafka process and elect a leader using the Raft protocol. See Section 15 for what to do if you really need ZooKeeper for a legacy reason.

🛒 GROCERY · The intercom now works without a separate switchboard operator. It calls its own shots using a vote.

07 Ansible Installs NiFi

NiFi 2.9.0 runs as a 3-node cluster. The playbook installs Java 21, pulls NiFi, configures TLS using certs from AWS ACM, then starts the cluster.

ansible-playbook -i inventory/aws_ec2.yml playbooks/nifi.yml

After the playbook finishes, open the NiFi UI at https://nifi.your-domain.com/. Login goes through your identity provider via OIDC.

🛒 GROCERY · The stock clerks are now in uniform, with name badges, and an org chart. Time to give them a procedure manual (the flow).

08 Upload Form & Text Extraction

A static React form is hosted in an S3 bucket behind CloudFront. When a user picks a file:

  1. 1. Form calls API Gateway → Lambda → returns a pre-signed S3 PUT URL
  2. 2. Browser uploads the file directly to S3 (raw bucket)
  3. 3. S3 event → Lambda extractor → calls Textract for PDFs/images, mammoth for DOCX
  4. 4. Extracted text is published to Kafka topic documents.extracted
  5. 5. NiFi consumes that topic and indexes the text into OpenSearch + writes the metadata row into Postgres

The form file is at html/upload.html — see the file index below.

🛒 GROCERY · A vendor backs the truck up to the dock, drops off a sealed box. The price-tag printer prints labels for what's inside. The intercom announces "New olives in aisle 4." The kiosk now finds them when customers ask.

09 Wire It All Together

Import the pre-built NiFi flow template. It contains processors for:

  • → ConsumeKafka_4_0 from documents.extracted
  • → JoltTransformJSON to shape the record
  • → PutOpenSearchHttpRecord to index it
  • → PutDatabaseRecord to write metadata to Postgres
  • → PutS3Object to archive the cleaned text

15 — About ZooKeeper (Important Honest Note)

You asked for "Kafka and ZooKeeper." Here's the honest 2026 reality.

Apache Kafka 4.0 (released March 2025) completely removed ZooKeeper support. The current 4.2.0 release (February 2026) only runs in KRaft mode — Kafka manages its own metadata using a built-in Raft consensus protocol. This guide uses KRaft because that's the modern, supported path.

If you have a hard requirement to use ZooKeeper anyway (legacy producers, an old Kafka Connect cluster you can't move yet), you have two options:

  1. Option A — Pin to Kafka 3.9.x. This was the last ZooKeeper-supporting release. Change kafka_version: "3.9.1" in ansible/roles/kafka/defaults/main.yml and uncomment the playbooks/zookeeper.yml play. You'll get bug fixes only — no new features.
  2. Option B — Run a separate ZooKeeper just for legacy clients. Keep Kafka on 4.2 (KRaft) and run a standalone ZooKeeper cluster elsewhere for any non-Kafka services that still need it (e.g. legacy HBase). Use the optional playbooks/zookeeper-standalone.yml play.

Recommendation: stay on KRaft. The migration path back is painful and the entire Kafka ecosystem has moved on.
