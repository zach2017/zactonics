Ansible File Operations

Comprehensive reference for file copy, existence checks, backup strategies, and common shell actions. All modules are ansible.builtin unless noted.

6 Copy entries 8 Existence entries 8 Backup entries 12 Shell entries 24 Quick-ref tasks

Copy a file from the control node (or remote) to a remote host.

- name: Copy config file
  ansible.builtin.copy:
    src: /local/path/app.conf
    dest: /etc/app/app.conf
    owner: root
    group: root
    mode: '0644'

Set to true to copy a file already on the remote host (not the controller).

- ansible.builtin.copy:
    src: /tmp/source.txt
    dest: /etc/dest.txt
    remote_src: true

Use rsync to sync files/directories efficiently between hosts. Requires ansible.posix collection.

- name: Sync app directory
  ansible.posix.synchronize:
    src: /src/app/
    dest: /opt/app/
    delete: true
    recursive: true

Fetch a file FROM a remote host back to the control node. Opposite direction of copy.

- name: Fetch log from remote
  ansible.builtin.fetch:
    src: /var/log/app.log
    dest: /tmp/fetched/
    flat: true

Copy a Jinja2 template, rendering variables before writing to the remote host.

- name: Deploy rendered config
  ansible.builtin.template:
    src: templates/nginx.conf.j2
    dest: /etc/nginx/nginx.conf
    mode: '0644'

Assemble a file on the remote from multiple fragment files in a source directory.

- ansible.builtin.assemble:
    src: /etc/motd.d/
    dest: /etc/motd

Gather facts about a file or path — existence, type, size, checksum, permissions, and more.

- name: Check if file exists
  ansible.builtin.stat:
    path: /etc/app/config.yml
  register: config_stat

- name: Act only if file exists
  debug:
    msg: "File found"
  when: config_stat.stat.exists

Boolean returned by the stat module — true if the path exists on the remote host.

when: config_stat.stat.exists

True if the path is a regular file — not a directory, symlink, or special file.

when: config_stat.stat.isreg

True if the path is a directory.

when: config_stat.stat.isdir

File size in bytes (valid only when stat.stat.exists is true). SHA1 checksum useful for detecting changes.

when: config_stat.stat.size > 0

- debug:
    var: config_stat.stat.checksum

Search for files matching patterns, age, size, or type within a path. Returns a list of matched paths.

- ansible.builtin.find:
    paths: /var/log/app/
    patterns: "*.log"
    age: "7d"
    recurse: true
  register: old_logs

Control when a task is marked failed or changed — useful after shell existence checks where non-zero rc is acceptable.

- shell: test -f /etc/myapp.conf
  register: result
  failed_when: result.rc not in [0, 1]
  changed_when: false

Auto-create a timestamped backup when copy would overwrite a file. Backup format: /path/file.YYYY-MM-DD@HH:MM:SS~

- ansible.builtin.copy:
    src: /tmp/new_app.log
    dest: /var/log/app.log
    backup: true

Same backup: true option works identically in the template module.

- ansible.builtin.template:
    src: app.conf.j2
    dest: /etc/app/app.conf
    backup: true

backup: true also available in lineinfile and blockinfile.

- ansible.builtin.lineinfile:
    path: /etc/app/app.conf
    regexp: '^log_level'
    line: 'log_level = INFO'
    backup: true

Manually copy a file to a backup path before overwriting, using a Unix timestamp suffix for uniqueness.

- name: Backup log before rotation
  ansible.builtin.copy:
    src: /var/log/app.log
    dest: "/var/log/app.log.{{ ansible_date_time.epoch }}.bak"
    remote_src: true

- name: Overwrite the original
  ansible.builtin.copy:
    src: /tmp/new_app.log
    dest: /var/log/app.log

Built-in facts — epoch gives Unix timestamp, date gives YYYY-MM-DD. Both ideal for unique backup suffixes.

dest: "/var/log/app.log.{{ ansible_date_time.epoch }}.bak"
dest: "/backup/app.log.{{ ansible_date_time.date }}.bak"

Compress files into .tar.gz / .zip archives for structured backups. Requires community.general collection.

- community.general.archive:
    path: /var/log/app/
    dest: /backup/app_logs_{{ ansible_date_time.date }}.tar.gz
    format: gz

Run a shell command with full shell features — pipes |, redirects >, globs *. Use when command is not sufficient.

- ansible.builtin.shell: |
    cat /var/log/app.log | grep ERROR > /tmp/errors.txt

Run a command without invoking a shell — safer and more idempotent. No pipes, redirects, or globs. Prefer over shell when possible.

- ansible.builtin.command:
    cmd: /usr/bin/myapp --check
    chdir: /opt/myapp

Send a raw SSH command — bypasses Python entirely. Used for bootstrapping when Python is not yet installed on the target.

- ansible.builtin.raw: apt-get install -y python3

Save a task's output — stdout, stderr, rc — into a variable for later use in the playbook.

- shell: cat /etc/os-release
  register: os_info

- debug:
    var: os_info.stdout_lines

creates: skip if the file exists. removes: skip if the file does not exist. Provides idempotency for command/shell tasks.

- ansible.builtin.command:
    cmd: /opt/setup.sh
    creates: /opt/.setup_done

- ansible.builtin.command:
    cmd: /opt/cleanup.sh
    removes: /opt/.setup_done

Ensure a specific line exists (or does not) in a file. Idempotent sed-like editing — will not duplicate lines.

- ansible.builtin.lineinfile:
    path: /etc/hosts
    line: '10.0.0.5 myserver'
    state: present

Insert or update a block of multiple lines wrapped by named marker comments to prevent duplication.

- ansible.builtin.blockinfile:
    path: /etc/ssh/sshd_config
    block: |
      AllowUsers deploy
      PermitRootLogin no
    marker: "# {mark} ANSIBLE MANAGED BLOCK"

Regex-based find-and-replace within a file. More powerful than lineinfile for pattern-based substitutions.

- ansible.builtin.replace:
    path: /etc/app/config.ini
    regexp: 'debug=true'
    replace: 'debug=false'

Create, delete, or set permissions/ownership on files, directories, or symlinks.

- ansible.builtin.file:
    path: /var/log/app
    state: directory
    owner: appuser
    mode: '0755'

- ansible.builtin.file:
    path: /tmp/stale.lock
    state: absent

Manage cron jobs — schedule shell actions to run periodically on remote hosts.

- ansible.builtin.cron:
    name: "Rotate app log"
    minute: "0"
    hour: "3"
    job: "/usr/sbin/logrotate /etc/logrotate.d/app"

Run a local script on the remote host — copies it first, then executes it. Supports creates for idempotency.

- ansible.builtin.script:
    cmd: scripts/init.sh
    creates: /opt/.initialized
