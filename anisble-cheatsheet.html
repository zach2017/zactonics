Level 1 — The Basics

The five modules you'll use every single day.

ansible.builtin.debug

docs ↗

Print messages or variable values during a run — your print() for playbooks.

Context: Controlled by -v verbosity via the verbosity param. Use var: to dump a variable (no Jinja braces) and msg: for formatted strings. Never put secrets in debug output — it lands in logs.
🟢 Simple 
- name: Say hello
  ansible.builtin.debug:
    msg: "Hello from {{ inventory_hostname }}"
🟡 Complete 
- name: Dump structured facts only when very verbose
  ansible.builtin.debug:
    var: ansible_facts.network_interfaces
    verbosity: 2          # shows only with -vv

- name: Conditional formatted message
  ansible.builtin.debug:
    msg: >-
      Host {{ inventory_hostname }} has
      {{ ansible_facts.processor_vcpus }} vCPUs and
      {{ (ansible_facts.memtotal_mb / 1024) | round(1) }} GB RAM
  when: ansible_facts.memtotal_mb is defined

ansible.builtin.command

docs ↗

Run a binary directly (no shell). Safer than shell — no pipes, redirects, or env expansion.

Context: Not idempotent by itself — guard it with creates:/removes: or when:. If you need |, >, && or $VARS, use ansible.builtin.shell instead.
🟢 Simple 
- name: Check version
  ansible.builtin.command: cat /etc/os-release
🟡 Complete 
- name: Run migration once (idempotent guard)
  ansible.builtin.command:
    cmd: /opt/app/bin/migrate --apply
    chdir: /opt/app
    creates: /opt/app/.migrated      # skips if file exists
  register: migrate_out
  changed_when: "'applied' in migrate_out.stdout"
  failed_when: migrate_out.rc not in [0, 2]

ansible.builtin.copy

docs ↗

Copy files to remote hosts, set ownership/permissions, optionally back up.

Context: For dynamic content use template. Use content: to write inline strings. backup: true keeps a timestamped copy before overwriting — great for configs.
🟢 Simple 
- name: Drop a config file
  ansible.builtin.copy:
    src: app.conf
    dest: /etc/app/app.conf
🟡 Complete 
- name: Deploy secured config with backup & validation
  ansible.builtin.copy:
    src: nginx.conf
    dest: /etc/nginx/nginx.conf
    owner: root
    group: root
    mode: "0640"
    backup: true
    validate: "nginx -t -c %s"   # %s = temp path, fails task if invalid
  notify: reload nginx

Level 2 — Files, Packages & Services

Configuration management core.

ansible.builtin.file

docs ↗

Manage paths: create dirs, symlinks, set perms, or delete with state: absent.

Context: state options: file, directory, link, hard, touch, absent. To create parent dirs recursively, use state: directory + recurse: true for perms.
🟢 Simple 
- name: Ensure log dir exists
  ansible.builtin.file:
    path: /var/log/app
    state: directory
🟡 Complete 
- name: Create owned tree + symlink
  ansible.builtin.file:
    path: /opt/app/releases
    state: directory
    owner: appuser
    group: appgrp
    mode: "0755"
    recurse: true

- name: Point 'current' at latest release
  ansible.builtin.file:
    src: /opt/app/releases/v2.4.1
    dest: /opt/app/current
    state: link
    force: true

ansible.builtin.package

docs ↗

OS-agnostic package install — auto-picks apt/yum/dnf/zypper.

Context: Great for cross-distro roles. When you need distro-specific options (e.g. apt's update_cache), use ansible.builtin.apt/dnf directly.
🟢 Simple 
- name: Install git
  ansible.builtin.package:
    name: git
    state: present
🟡 Complete 
- name: Install a tool set across distros
  ansible.builtin.package:
    name:
      - curl
      - jq
      - python3-pip
    state: latest
  register: pkg
  retries: 3
  delay: 5
  until: pkg is succeeded

ansible.builtin.service

docs ↗

Start/stop/enable services across init systems.

Context: For systemd-only features (daemon-reload, masking) prefer ansible.builtin.systemd_service. Pair with handlers so restarts only fire on change.
🟢 Simple 
- name: Ensure nginx running
  ansible.builtin.service:
    name: nginx
    state: started
    enabled: true
🟡 Complete 
# handlers/main.yml
- name: reload nginx
  ansible.builtin.systemd_service:
    name: nginx
    state: reloaded
    daemon_reload: true

Level 3 — Control Flow & Templates

Loops, conditionals, dynamic content.

ansible.builtin.template

docs ↗

Render a Jinja2 .j2 file with variables and copy it to the host.

Context: Templates run on the controller. Add validate: for config safety and {{ ansible_managed }} headers so humans know not to hand-edit.
🟢 Simple 
- name: Render config
  ansible.builtin.template:
    src: app.conf.j2
    dest: /etc/app/app.conf
🟡 Complete 
- name: Render env-specific config
  ansible.builtin.template:
    src: "app.{{ env }}.conf.j2"
    dest: /etc/app/app.conf
    mode: "0640"
    backup: true
    validate: "/usr/bin/appctl check %s"
  vars:
    db_pool: "{{ 50 if env == 'prod' else 5 }}"
  notify: restart app

ansible.builtin.set_fact + loops

docs ↗

Compute and store variables mid-play; iterate with loop.

Context: set_fact values persist for the host across the play. Prefer loop: over the legacy with_*. Use loop_control: label: to keep output readable.
🟢 Simple 
- name: Build a fact
  ansible.builtin.set_fact:
    app_url: "https://{{ inventory_hostname }}:8443"
🟡 Complete 
- name: Create users from a list
  ansible.builtin.user:
    name: "{{ item.name }}"
    groups: "{{ item.groups }}"
    state: present
  loop: "{{ team_members }}"
  loop_control:
    label: "{{ item.name }}"
  when: item.active | default(true)

ansible.builtin.uri

docs ↗

Call HTTP/REST APIs — health checks, webhooks, ETL ingestion.

Context: Set return_content: true to capture the body, and combine with retries/until for polling. status_code: can be a list of acceptable codes.
🟢 Simple 
- name: Health check
  ansible.builtin.uri:
    url: https://app.local/healthz
    status_code: 200
🟡 Complete 
- name: POST JSON and poll until ready
  ansible.builtin.uri:
    url: https://api.local/v1/jobs
    method: POST
    headers:
      Authorization: "Bearer {{ api_token }}"
    body_format: json
    body: { type: "etl", source: "s3://bucket/raw" }
    return_content: true
    status_code: [200, 201, 202]
  register: job
  retries: 10
  delay: 6
  until: job.json.status == "completed"

Level 4 — Advanced Orchestration

Blocks, async, error handling, delegation.

block / rescue / always

docs ↗

Group tasks with try/catch/finally semantics.

Context: rescue runs on failure, always runs regardless — perfect for cleanup, rollbacks, and notifications in ETL/DevOps pipelines.
🟡 Complete 
- name: Safe deploy with rollback
  block:
    - ansible.builtin.command: /opt/app/deploy.sh {{ version }}
    - ansible.builtin.uri:
        url: https://app.local/healthz
        status_code: 200
  rescue:
    - ansible.builtin.debug:
        msg: "Deploy failed — rolling back"
    - ansible.builtin.command: /opt/app/rollback.sh
  always:
    - ansible.builtin.command: /opt/app/notify.sh "{{ ansible_failed_result | default('ok') }}"

async / poll + wait_for

docs ↗

Run long jobs without blocking; wait for ports/files.

Context: poll: 0 fires-and-forgets; check later with async_status. wait_for is ideal for "service is up on port X" gates.
🟡 Complete 
- name: Kick off long ETL batch (fire and forget)
  ansible.builtin.command: /opt/etl/run_batch.sh
  async: 3600
  poll: 0
  register: batch

- name: Wait for results port to open
  ansible.builtin.wait_for:
    host: "{{ inventory_hostname }}"
    port: 9000
    timeout: 300

- name: Reap the async job
  ansible.builtin.async_status:
    jid: "{{ batch.ansible_job_id }}"
  register: r
  until: r.finished
  retries: 60
  delay: 30

Level 5 — AWS & Azure

Cloud modules live in amazon.aws & azure.azcollection, driven by builtin control flow.

amazon.aws.s3_object

docs ↗

Upload/download/list S3 objects — the backbone of ETL ingest/landing zones.

Context: Install with ansible-galaxy collection install amazon.aws and pip install boto3 botocore. Prefer IAM roles / env creds over hard-coded keys.
🟢 Simple 
- name: Upload a file to S3
  amazon.aws.s3_object:
    bucket: my-data-lake
    object: /raw/events.json
    src: /tmp/events.json
    mode: put
🟡 Complete 
- name: Launch EC2 worker + land data in S3
  amazon.aws.ec2_instance:
    name: "etl-worker-{{ item }}"
    instance_type: t3.large
    image_id: "{{ ami_id }}"
    region: us-east-1
    security_groups: [etl-sg]
    tags: { role: etl, env: prod }
    wait: true
  loop: "{{ range(1, worker_count + 1) | list }}"
  register: workers

- name: Sync processed output to S3
  amazon.aws.s3_object:
    bucket: my-data-lake
    object: "/processed/{{ run_id }}/part-{{ item }}.parquet"
    src: "/data/out/part-{{ item }}.parquet"
    mode: put
    encrypt: true
  loop: "{{ range(0, partitions) | list }}"

amazon.aws.rds_instance

docs ↗

Provision managed databases for ETL targets/warehouses.

Context: Pull the password from a vault/secret, never inline. Pair with community.aws.rds_instance_info to wait for availability.
🟡 Complete 
- name: Provision PostgreSQL warehouse
  amazon.aws.rds_instance:
    id: dw-prod
    engine: postgres
    db_instance_class: db.r6g.xlarge
    allocated_storage: 200
    master_username: dw_admin
    master_user_password: "{{ vault_dw_password }}"
    region: us-east-1
    tags: { team: data, tier: warehouse }
    wait: true

azure.azcollection.azure_rm_storageblob

docs ↗

Upload/download blobs in Azure Storage — Azure's ETL landing zone.

Context: Install with ansible-galaxy collection install azure.azcollection and its requirements. Authenticate via az login, service principal env vars, or managed identity.
🟢 Simple 
- name: Upload a blob
  azure.azcollection.azure_rm_storageblob:
    resource_group: data-rg
    storage_account_name: mydatalake
    container: raw
    blob: events.json
    src: /tmp/events.json
🟡 Complete 
- name: Ensure container then push processed files
  azure.azcollection.azure_rm_storageblob:
    resource_group: data-rg
    storage_account_name: mydatalake
    container: processed
    blob: "{{ run_id }}/part-{{ item }}.parquet"
    src: "/data/out/part-{{ item }}.parquet"
    content_type: application/octet-stream
  loop: "{{ range(0, partitions) | list }}"
  loop_control:
    label: "part-{{ item }}"

azure.azcollection.azure_rm_virtualmachine

docs ↗

Create VMs to run pipelines or services.

Context: Use SSH keys not passwords; tag for cost tracking; combine with azure_rm_postgresqlflexibleserver for managed DB targets.
🟡 Complete 
- name: Provision ETL worker VM
  azure.azcollection.azure_rm_virtualmachine:
    resource_group: data-rg
    name: "etl-worker-{{ item }}"
    vm_size: Standard_D4s_v5
    admin_username: azureuser
    ssh_password_enabled: false
    ssh_public_keys:
      - path: /home/azureuser/.ssh/authorized_keys
        key_data: "{{ ssh_pub_key }}"
    image:
      offer: 0001-com-ubuntu-server-jammy
      publisher: Canonical
      sku: 22_04-lts
      version: latest
    tags: { role: etl, env: prod }
  loop: "{{ range(1, worker_count + 1) | list }}"

Level 6 — Data ETL Apps & Services

Builtins gluing Extract → Transform → Load.

End-to-end ETL with builtins

docs ↗

Extract via get_url/uri, transform via command, load via cloud modules — with idempotent guards.

Context: Keep each stage idempotent. Use register + changed_when so reruns are safe. Stamp a run_id with set_fact for traceable artifact paths.
🟡 Complete 
- name: Stamp run id
  ansible.builtin.set_fact:
    run_id: "{{ ansible_date_time.iso8601_basic_short }}"

# EXTRACT
- name: Pull source dataset
  ansible.builtin.get_url:
    url: "https://feeds.example.com/{{ dataset }}.csv"
    dest: "/data/raw/{{ run_id }}.csv"
    mode: "0644"
    checksum: "sha256:{{ expected_sha }}"

# TRANSFORM
- name: Run transform job
  ansible.builtin.command:
    cmd: "python3 transform.py /data/raw/{{ run_id }}.csv /data/out/{{ run_id }}.parquet"
    creates: "/data/out/{{ run_id }}.parquet"
  register: tf
  changed_when: tf.rc == 0

# LOAD (AWS)
- name: Load to data lake
  amazon.aws.s3_object:
    bucket: my-data-lake
    object: "/curated/{{ run_id }}.parquet"
    src: "/data/out/{{ run_id }}.parquet"
    mode: put
    encrypt: true

# VALIDATE
- name: Notify warehouse to refresh
  ansible.builtin.uri:
    url: https://warehouse.local/v1/refresh
    method: POST
    body_format: json
    body: { run_id: "{{ run_id }}" }
    status_code: 202

Level 7 — Logging & DevOps

Observability, CI/CD glue, audit trails.

lineinfile · blockinfile · log capture

docs ↗

Manage log config, ship run output, append audit records idempotently.

Context: Use blockinfile's managed markers for editable config sections; lineinfile for single directives. For full files always prefer template.
🟢 Simple 
- name: Ensure log level
  ansible.builtin.lineinfile:
    path: /etc/app/app.conf
    regexp: '^log_level='
    line: 'log_level=info'
🟡 Complete 
- name: Configure rsyslog forwarding block
  ansible.builtin.blockinfile:
    path: /etc/rsyslog.d/50-forward.conf
    create: true
    marker: "# {mark} ANSIBLE MANAGED — log shipping"
    block: |
      *.* action(type="omfwd"
        target="logs.internal" port="514" protocol="tcp")
  notify: restart rsyslog

- name: Append deploy audit record
  ansible.builtin.lineinfile:
    path: /var/log/deploys.log
    create: true
    line: "{{ ansible_date_time.iso8601 }} {{ version }} by {{ lookup('env','USER') }}"
    insertafter: EOF

- name: Capture & archive command output
  ansible.builtin.command: /opt/ci/run_tests.sh
  register: ci
  changed_when: false
- ansible.builtin.copy:
    content: "{{ ci.stdout }}\n--- rc={{ ci.rc }} ---"
    dest: "/var/log/ci/{{ run_id | default('latest') }}.log"
★

Best Practices for Builtins

1. Always FQCN

Use ansible.builtin.copy, not copy. Prevents collection collisions and is lint-clean.

2. Idempotency first

Guard command/shell with creates, removes, when, or changed_when. Reruns must be no-ops.

3. command > shell

Only use shell when you truly need pipes/redirects/expansion. command avoids injection risk.

4. Name every task

Readable output and audit logs. Names double as documentation.

5. Handlers for restarts

notify + handlers means services restart only when config actually changes.

6. Secrets in Vault

Never inline keys/passwords. Use ansible-vault or cloud secret stores; pass via vars.

7. loop, not with_*

Modern loop: + loop_control: label: keeps logs clean and behavior predictable.

8. Validate before apply

Use validate: on copy/template to fail safely on bad config.

9. check_mode & --diff

Dry-run with --check --diff in CI before real applies; mark read-only tasks check_mode: false where needed.

10. Retry transient ops

Network/cloud calls: retries + delay + until for resilience.

11. block/rescue for safety

Wrap risky deploys with rollback in rescue and cleanup in always.

12. Don't log secrets

Add no_log: true on tasks handling sensitive data.
