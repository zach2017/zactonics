01

Verify Disk Space & RAM

PRE-REQ

Check File Explorer and Task Manager before downloading

Check disk space:

  1. Press Win + E to open File Explorer.
  2. Click This PC in the left panel.
  3. Check the bar under Local Disk (C:) — you need at least 30 GB free.
  4. If short, open Settings → System → Storage → Storage Sense and free space first.

Check RAM:

  1. Press Ctrl + Shift + Esc to open Task Manager.
  2. Click the Performance tab, then click Memory.
  3. Confirm total RAM is 8 GB or more.
  4. Confirm Available memory is at least 4 GB. If not, close other apps first.
Tip: With 8 GB RAM, the model occupies nearly all available memory. Keep Task Manager open to monitor RAM during sessions.
02

Download & Install Ollama

INSTALL

The AI model engine — runs silently in background

  1. Open your browser and go to https://ollama.com/download
  2. Click Download for Windows. The file OllamaSetup.exe downloads (~80 MB).
  3. Open your Downloads folder and double-click OllamaSetup.exe.
  4. If Windows shows a blue "protected your PC" warning, click More info → Run anyway.
  5. Follow installer prompts and accept all defaults.
  6. After install, look for a small llama icon in the system tray (bottom-right near the clock). That confirms Ollama is running.
Note: Ollama starts automatically every time Windows boots. You don't need to re-launch it manually.
03

Open a Command Prompt

INSTALL

One-time step — needed to download the model

  1. Press Win + S (or click the Search bar on the taskbar).
  2. Type cmd and press Enter.
  3. A black Command Prompt window opens. Leave it open for Step 4.
Note: You do not need to run as administrator. A standard Command Prompt is sufficient.

05

Install Docker Desktop

INSTALL

Required to run Open WebUI browser interface

  1. Go to https://www.docker.com/products/docker-desktop
  2. Click Download for Windows.
  3. Run the installer and accept all defaults.
  4. Restart Windows when prompted. Docker Desktop opens automatically after restart.
  5. Wait for Docker Desktop to show a green "Engine running" indicator (bottom-left). This takes 1–2 minutes.
8 GB RAM caution: Docker Desktop uses 500 MB–1 GB RAM. Combined with the Ollama model you will be near your memory limit. Close all other applications during sessions.
06

Install Open WebUI

INSTALL

Browser-based chat interface — one command

  1. Press Win + S, type cmd, press Enter to open a new Command Prompt.
  2. Copy the entire command below (one long line — do not split it) and paste it, then press Enter.
Command Prompt — paste entire line 
docker run -d -p 3000:8080 --add-host=host.docker.internal:host-gateway -v open-webui:/app/backend/data --name open-webui --restart always ghcr.io/open-webui/open-webui:main
  1. Docker downloads Open WebUI (~1 GB). Wait until the Command Prompt returns to a blinking cursor.
  2. Open WebUI is now running and starts automatically with Windows + Docker going forward.
Important: That command is a single line. Copy it in full. If you receive an error, confirm Docker Desktop shows Engine running (green) and try again.
07

Open Browser & Connect Model

CONFIGURE

Create local account · select model · bookmark the URL

  1. Open any browser (Edge, Chrome, Firefox) and go to http://localhost:3000
  2. Click Sign up and create a local admin account. Use any email and password — stored locally only, never transmitted.
  3. After logging in, click the model selector dropdown (top-left of the screen).
  4. Select ALIENTELLIGENCE/aipoweredlawfirms from the list.
  5. If the model doesn't appear, click the refresh icon next to the model selector to re-scan Ollama.
Bookmark this address: Add http://localhost:3000 to your browser favorites. Works only on your laptop — not accessible from the internet.

Pre-Flight Help

Why 8 GB RAM is the minimum: The legal model requires approximately 4–5 GB of RAM in 4-bit quantized form. The remaining RAM is needed by Windows, Docker, and your browser. Running below 8 GB will cause out-of-memory crashes.

How to free up disk space: Open Settings → System → Storage, click "Temporary files" and select items to delete. Empty the Recycle Bin. Uninstall unused programs via Control Panel.

What 4-bit quantization means: To fit the model into 8 GB RAM, Ollama automatically loads it in 4-bit precision (Q4_K_M). This reduces memory usage at a small cost to response quality. For legal drafting and analysis the quality difference is not significant in practice.

ABA Rule 1.6 note: Running the AI locally means client data never leaves your machine. No third-party vendor can access prompts or responses. This satisfies the confidentiality requirements under ABA Model Rule 1.6 and most state bar technology competence opinions.

Step 1 — More Help

How to check disk space visually: In File Explorer → This PC, the colored bar under your drive turns red when space is critically low. Hover over the drive icon to see exact free/total GB.

Alternative: use the Run dialog. Press Win + R, type dfrgui, press Enter to open Disk Defragmenter and see drive sizes.

RAM isn't showing 8 GB? In Task Manager → Performance → Memory, the number shown is the total installed RAM. If it shows 4 GB you may have 2 sticks of RAM with one slot unused — adding another 4 GB stick would bring you to 8 GB. Consult your laptop's manual for upgradability.

What if Available RAM is under 4 GB? Close your browser, email client, and any other open apps. Background services like OneDrive, antivirus, and Teams consume RAM even when idle. Check Task Manager → Processes and end any non-essential tasks.

Step 2 — Ollama Help

What is Ollama? Ollama is an open-source tool that manages AI model downloads, storage, and inference. It runs as a local server on port 11434. When you open the WebUI or a terminal prompt, it talks to Ollama in the background. Think of it like a local API gateway for AI models.

"Windows protected your PC" warning: This is a standard Windows SmartScreen warning for newly downloaded executables. Ollama is an established, widely-used open-source tool. Click "More info" then "Run anyway" to proceed. You can verify the download hash on the Ollama GitHub releases page.

Can't find the system tray icon? Click the small arrow (^) in the bottom-right corner of your taskbar to expand the hidden icons area. The llama icon should appear there. If it's absent, Ollama may not have launched — try running it from the Start menu.

Where does Ollama store models? Models are stored in C:\Users\YourName\.ollama\models. You can change this by setting the environment variable OLLAMA_MODELS to a different path before installing.

Step 3 — Command Prompt Help

What is Command Prompt? Command Prompt (cmd) is a text-based interface built into Windows. You type commands and Windows executes them. You only need it once — to download the AI model in Step 4. After that, daily use is entirely browser-based.

Alternative: use Windows Terminal or PowerShell. Both work equally well. Search for "Windows Terminal" or "PowerShell" and use those if you prefer them over Command Prompt.

Right-click to paste in Command Prompt. Unlike most Windows apps, Command Prompt doesn't accept Ctrl+V by default. Right-click inside the black window to paste a command. Or enable Ctrl+V by right-clicking the title bar → Properties → Options → Enable Ctrl key shortcuts.

Step 4 — Model Download Help

About the ALIENTELLIGENCE/aipoweredlawfirms model: This is a community-published model on Ollama's model registry, built on Meta's Llama 3.1 architecture with specialized system instructions for legal analysis. It is designed for contract review, document summarization, legal research, and correspondence drafting.

Why does it take so long to start after downloading? On CPU-only with 8 GB RAM, the model loads from disk into memory each time you send the first message. This "cold start" takes 30–90 seconds. Subsequent messages in the same session are faster because the model stays in RAM.

Download interrupted? Run ollama run ALIENTELLIGENCE/aipoweredlawfirms again. Ollama tracks progress and resumes from the last completed chunk — you won't re-download what already arrived.

Want to verify the model afterward? Open a new Command Prompt and type ollama list. You should see the aipoweredlawfirms model in the output with its size.

Step 5 — Docker Help

Why is Docker needed? Open WebUI is distributed as a Docker container — a self-contained package with all its dependencies. Docker Desktop runs the container on your Windows machine without requiring manual installation of Python, Node.js, or other runtimes.

Alternative: skip Docker entirely. If Docker uses too much RAM on your 8 GB system, you can interact with the model directly via the Command Prompt chat that appears after running ollama run ALIENTELLIGENCE/aipoweredlawfirms. It's text-only but functional for short queries.

WSL2 requirement: Docker Desktop on Windows requires WSL2 (Windows Subsystem for Linux 2). The Docker installer handles this automatically on Windows 10 version 2004 and later. If your Windows is older, you may need to enable WSL manually first.

Docker Desktop uses RAM: By default Docker Desktop allocates memory dynamically. On 8 GB systems, consider limiting its memory in Docker Desktop → Settings → Resources → Advanced → Memory. Set to 1 GB to reserve more for the model.

Step 6 — Open WebUI Help

What is Open WebUI? Open WebUI is an open-source, browser-based interface for local AI models. It provides a chat experience similar to ChatGPT but runs entirely on your machine. It supports document uploads, conversation history, model switching, and system prompt customization.

The docker command explained:

  • -d — run in background (detached)
  • -p 3000:8080 — map port 3000 on your PC to port 8080 in the container
  • --add-host=host.docker.internal:host-gateway — lets WebUI talk to Ollama on the same machine
  • -v open-webui:/app/backend/data — save your conversations persistently
  • --restart always — start automatically with Docker

Container already exists error? If you see "container name already in use," run: docker rm open-webui then run the install command again.

Step 7 — Browser Interface Help

The sign-up screen asks for an email — is that sent anywhere? No. Open WebUI stores your credentials in a local SQLite database on your machine at C:\Users\YourName\AppData\Local\... via the Docker volume. No email confirmation is sent; no account is created on any external service.

Model doesn't appear in the dropdown? Click the circular refresh icon next to the model selector. If it still doesn't appear, open Command Prompt and confirm Ollama is running by typing ollama list. You should see the model listed.

Can I use Open WebUI from another device on my network? Yes. Other devices can access it at http://[your-laptop-IP]:3000. However, for confidentiality the Zactonics AI recommendation is to leave it accessible only from localhost (the default).

How to upload a document: In the chat box, click the paperclip icon (or + icon depending on version) to upload a file. Paste long documents as text into the chat box for fastest results on CPU-only hardware.

Step 8 — Testing & Troubleshooting

Response never starts — what to check:

  1. Is the Ollama llama icon visible in the system tray? If not, search Start for "Ollama" and launch it.
  2. Is Docker Desktop open and showing a green "Engine running" indicator?
  3. Is the correct model selected in the Open WebUI dropdown?
  4. Do you have at least 4 GB of Available RAM? (Task Manager → Performance → Memory)
  5. Try refreshing the browser page and sending the message again.

Response starts but is garbled or extremely slow: This can happen if Windows is swapping memory to disk (paging). Close all other applications and try again. If the problem persists, restart Windows to clear memory pressure.

Good sample legal prompts to test with:

  • What are the elements of a valid contract under common law?
  • Draft a short engagement letter for a new client matter.
  • Identify three red flags in this clause: [paste clause text]
