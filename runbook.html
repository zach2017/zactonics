00 — Overview

From a bare Linux box to a working Kafka cluster

This runbook takes you from nothing—a freshly imaged Ubuntu 22.04 workstation—through installing every tool, configuring AWS, authoring the CloudFormation template, driving the whole deployment with Ansible, running health checks, and finally proving the cluster works with a Python producer/consumer. Each block is copy-paste runnable.

04 — CloudFormation

Author the cluster template

Save as cloudformation/kafka-cluster.yaml. Defines VPC, 3 subnets, security group, and the MSK cluster.

AWSTemplateFormatVersion: '2010-09-09'
Description: Amazon MSK Kafka Cluster (3 brokers / 3 AZ)

Parameters:
  ClusterName: { Type: String, Default: my-kafka-cluster }
  KafkaVersion: { Type: String, Default: 3.6.0 }
  InstanceType: { Type: String, Default: kafka.m5.large }

Resources:
  KafkaVPC:
    Type: AWS::EC2::VPC
    Properties: { CidrBlock: 10.0.0.0/16, EnableDnsSupport: true, EnableDnsHostnames: true }

  Subnet1:
    Type: AWS::EC2::Subnet
    Properties: { VpcId: !Ref KafkaVPC, CidrBlock: 10.0.1.0/24, AvailabilityZone: !Select [0, !GetAZs ''] }
  Subnet2:
    Type: AWS::EC2::Subnet
    Properties: { VpcId: !Ref KafkaVPC, CidrBlock: 10.0.2.0/24, AvailabilityZone: !Select [1, !GetAZs ''] }
  Subnet3:
    Type: AWS::EC2::Subnet
    Properties: { VpcId: !Ref KafkaVPC, CidrBlock: 10.0.3.0/24, AvailabilityZone: !Select [2, !GetAZs ''] }

  KafkaSecurityGroup:
    Type: AWS::EC2::SecurityGroup
    Properties:
      GroupDescription: Allow Kafka traffic
      VpcId: !Ref KafkaVPC
      SecurityGroupIngress:
        - { IpProtocol: tcp, FromPort: 9092, ToPort: 9098, CidrIp: 10.0.0.0/16 }

  KafkaCluster:
    Type: AWS::MSK::Cluster
    Properties:
      ClusterName: !Ref ClusterName
      KafkaVersion: !Ref KafkaVersion
      NumberOfBrokerNodes: 3
      BrokerNodeGroupInfo:
        InstanceType: !Ref InstanceType
        ClientSubnets: [ !Ref Subnet1, !Ref Subnet2, !Ref Subnet3 ]
        SecurityGroups: [ !Ref KafkaSecurityGroup ]
        StorageInfo: { EBSStorageInfo: { VolumeSize: 100 } }
      EncryptionInfo:
        EncryptionInTransit: { ClientBroker: TLS, InCluster: true }

Outputs:
  ClusterArn: { Value: !Ref KafkaCluster }
  VPCId: { Value: !Ref KafkaVPC }
05 — Ansible Automation

Orchestrate the deployment

Ansible wraps CloudFormation so the whole stack is one idempotent command. Save as playbooks/deploy.yml.

---
- name: Deploy Kafka (MSK) via CloudFormation
  hosts: localhost
  connection: local
  gather_facts: false
  vars:
    region: us-east-1
    stack_name: kafka-stack
    template: ../cloudformation/kafka-cluster.yaml
  tasks:
    - name: Validate template
      amazon.aws.cloudformation_info:
      register: _   # no-op presence check; real validate below
      ignore_errors: true

    - name: Create / update the CloudFormation stack
      amazon.aws.cloudformation:
        stack_name: "{{ stack_name }}"
        state: present
        region: "{{ region }}"
        template: "{{ template }}"
        capabilities: [ CAPABILITY_NAMED_IAM ]
        on_create_failure: ROLLBACK
      register: cfn

    - name: Show stack outputs
      ansible.builtin.debug:
        var: cfn.stack_outputs

    - name: Fetch MSK bootstrap brokers
      ansible.builtin.command: >
        aws kafka get-bootstrap-brokers
        --cluster-arn {{ cfn.stack_outputs.ClusterArn }}
        --region {{ region }} --output json
      register: brokers
      changed_when: false

    - name: Save broker string to a file for the client
      ansible.builtin.copy:
        dest: ../client/brokers.json
        content: "{{ brokers.stdout }}"

Run it:

cd playbooks
ansible-playbook deploy.yml

07 — Health Checks

Verify the cluster is healthy

Save as playbooks/healthcheck.yml — confirms ACTIVE state and broker count.

---
- name: Kafka cluster health checks
  hosts: localhost
  connection: local
  gather_facts: false
  vars: { region: us-east-1, stack_name: kafka-stack }
  tasks:
    - name: Get cluster ARN from stack
      ansible.builtin.shell: >
        aws cloudformation describe-stacks --stack-name {{ stack_name }}
        --query "Stacks[0].Outputs[?OutputKey=='ClusterArn'].OutputValue"
        --output text --region {{ region }}
      register: arn
      changed_when: false

    - name: Describe MSK cluster
      ansible.builtin.shell: >
        aws kafka describe-cluster --cluster-arn {{ arn.stdout }}
        --region {{ region }} --output json
      register: desc
      changed_when: false

    - name: Assert cluster is ACTIVE
      ansible.builtin.assert:
        that: "(desc.stdout | from_json).ClusterInfo.State == 'ACTIVE'"
        success_msg: "Cluster is ACTIVE ✓"
        fail_msg: "Cluster not active — check the console"

    - name: Report broker count
      ansible.builtin.debug:
        msg: "Brokers: {{ (desc.stdout | from_json).ClusterInfo.NumberOfBrokerNodes }}"

Manual one-liners for quick checks:

# Cluster state
aws kafka describe-cluster --cluster-arn $ARN \
  --query 'ClusterInfo.State' --output text

# List topics using Kafka CLI tools (run from inside the VPC)
bin/kafka-topics.sh --bootstrap-server $BROKERS --list

# TCP reachability to a broker
nc -zv broker1.kafka.amazonaws.com 9094
08 — Python Client

Send & receive messages

Round-trip test using kafka-python over TLS. Replace the broker string with your TLS endpoint (port 9094) from brokers.json.

#!/usr/bin/env python3
"""producer.py — publish messages to a TLS-secured MSK topic."""
import json, time
from kafka import KafkaProducer

BROKERS = ["b-1.my-kafka.xxxx.kafka.us-east-1.amazonaws.com:9094"]
TOPIC   = "demo-topic"

producer = KafkaProducer(
    bootstrap_servers=BROKERS,
    security_protocol="SSL",                       # TLS in transit
    value_serializer=lambda v: json.dumps(v).encode("utf-8"),
    acks="all",                                    # wait for all replicas
    retries=3,
)

for i in range(10):
    msg = {"id": i, "ts": time.time(), "payload": f"hello-{i}"}
    future = producer.send(TOPIC, value=msg)
    record = future.get(timeout=10)                # block to confirm
    print(f"sent id={i} -> partition {record.partition} offset {record.offset}")

producer.flush()
producer.close()
print("done.")
#!/usr/bin/env python3
"""consumer.py — subscribe and print messages from the topic."""
import json
from kafka import KafkaConsumer

BROKERS = ["b-1.my-kafka.xxxx.kafka.us-east-1.amazonaws.com:9094"]
TOPIC   = "demo-topic"

consumer = KafkaConsumer(
    TOPIC,
    bootstrap_servers=BROKERS,
    security_protocol="SSL",
    auto_offset_reset="earliest",                  # read from start
    group_id="demo-group",
    value_deserializer=lambda b: json.loads(b.decode("utf-8")),
    consumer_timeout_ms=15000,                      # stop after 15s idle
)

print("listening...")
for record in consumer:
    print(f"recv partition {record.partition} offset {record.offset}: {record.value}")
consumer.close()

Run both (consumer in one terminal, producer in another):

# terminal 1
python3 client/consumer.py
# terminal 2
python3 client/producer.py
Success looks like: the producer prints partition/offset confirmations and the consumer prints all 10 JSON messages. That round-trip proves the cluster accepts writes, replicates, and serves reads.

Tear down safely

⚠ MSK bills hourly. One Ansible command removes everything.

# via Ansible
ansible localhost -m amazon.aws.cloudformation \
  -a "stack_name=kafka-stack state=absent region=us-east-1"

# or raw CLI
aws cloudformation delete-stack --stack-name kafka-stack
aws cloudformation wait stack-delete-complete --stack-name kafka-stack
