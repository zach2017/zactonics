00 — Overview

Install Kafka onto raw EC2 with Ansible

This runbook builds a self-managed 3-broker Kafka cluster — you own the servers. Ansible provisions three EC2 instances from a base Amazon Linux 2023 AMI, then connects over SSH to install Java and Apache Kafka, render a unique server.properties per broker, and run each broker as a systemd service. Uses KRaft mode (no ZooKeeper).

02 — Provision EC2

Launch three base instances with Ansible

Save as playbooks/provision.yml. Creates a security group and three EC2 instances from the base Amazon Linux 2023 AMI.

---
- name: Provision Kafka EC2 instances
  hosts: localhost
  connection: local
  gather_facts: false
  vars:
    region: us-east-1
    keypair: kafka-key
    instance_type: t3.medium
    ami: ami-0abcdef1234567890   # Amazon Linux 2023 in your region
  tasks:
    - name: Security group for Kafka
      amazon.aws.ec2_security_group:
        name: kafka-sg
        description: Kafka self-managed
        region: "{{ region }}"
        rules:
          - { proto: tcp, ports: [22],          cidr_ip: 0.0.0.0/0 }   # SSH (lock down in prod)
          - { proto: tcp, ports: [9092,9093],   cidr_ip: 10.0.0.0/16 } # client + controller
      register: sg

    - name: Launch 3 broker instances
      amazon.aws.ec2_instance:
        name: "kafka-broker-{{ item }}"
        region: "{{ region }}"
        key_name: "{{ keypair }}"
        instance_type: "{{ instance_type }}"
        image_id: "{{ ami }}"
        security_group: "{{ sg.group_id }}"
        network: { assign_public_ip: true }
        tags: { role: kafka, broker_id: "{{ item }}" }
        wait: true
        count: 1
      loop: [1, 2, 3]
      register: ec2

In production you'd place these in private subnets behind a bastion; public IPs here keep the demo simple.

03 — Inventory

Dynamic inventory from EC2 tags

Save as inventory/aws_ec2.yml. Ansible discovers brokers by their role: kafka tag — no hand-editing IPs.

plugin: amazon.aws.aws_ec2
regions: [ us-east-1 ]
filters:
  tag:role: kafka
  instance-state-name: running
keyed_groups:
  - key: tags.role
    prefix: ""
    separator: ""
hostnames: [ ip-address ]
compose:
  ansible_host: public_ip_address
  broker_id: tags.broker_id

Group vars — inventory/group_vars/kafka.yml:

ansible_user: ec2-user
ansible_ssh_private_key_file: ~/.ssh/kafka-key.pem
ansible_ssh_common_args: '-o StrictHostKeyChecking=no'
kafka_version: "3.7.0"
kafka_scala: "2.13"

04a — Install Tasks

roles/kafka/tasks/main.yml

---
- name: Install Java 17
  ansible.builtin.dnf: { name: java-17-amazon-corretto-headless, state: present }
  become: true

- name: Create kafka system user
  ansible.builtin.user: { name: kafka, system: true, shell: /sbin/nologin }
  become: true

- name: Download Kafka
  ansible.builtin.get_url:
    url: "https://downloads.apache.org/kafka/{{ kafka_version }}/kafka_{{ kafka_scala }}-{{ kafka_version }}.tgz"
    dest: /tmp/kafka.tgz
  become: true

- name: Unpack to /opt/kafka
  ansible.builtin.unarchive:
    src: /tmp/kafka.tgz
    dest: /opt/
    remote_src: true
    creates: "/opt/kafka_{{ kafka_scala }}-{{ kafka_version }}"
  become: true

- name: Symlink /opt/kafka
  ansible.builtin.file:
    src: "/opt/kafka_{{ kafka_scala }}-{{ kafka_version }}"
    dest: /opt/kafka
    state: link
  become: true

- name: Create data + log dirs
  ansible.builtin.file: { path: "{{ item }}", state: directory, owner: kafka, group: kafka }
  loop: [ /var/lib/kafka, /var/log/kafka ]
  become: true

- name: Render server.properties
  ansible.builtin.template:
    src: server.properties.j2
    dest: /opt/kafka/config/kraft/server.properties
    owner: kafka
  become: true
  notify: restart kafka

- name: Format KRaft storage (idempotent)
  ansible.builtin.shell: |
    if [ ! -f /var/lib/kafka/meta.properties ]; then
      KAFKA_CLUSTER_ID=$(/opt/kafka/bin/kafka-storage.sh random-uuid)
      /opt/kafka/bin/kafka-storage.sh format -t $KAFKA_CLUSTER_ID \
        -c /opt/kafka/config/kraft/server.properties
    fi
  args: { creates: /var/lib/kafka/meta.properties }
  become: true

- name: Install systemd unit
  ansible.builtin.template: { src: kafka.service.j2, dest: /etc/systemd/system/kafka.service }
  become: true
  notify: restart kafka

- name: Enable + start Kafka
  ansible.builtin.systemd: { name: kafka, enabled: true, state: started, daemon_reload: true }
  become: true
04b — Broker Config

templates/server.properties.j2

Each broker gets a unique node.id and the full controller quorum list. Jinja2 fills these from inventory.

# --- KRaft combined broker+controller ---
process.roles=broker,controller
node.id={{ broker_id }}

# Quorum: every broker, built dynamically from the kafka group
controller.quorum.voters={% for h in groups['kafka'] %}{{ hostvars[h].broker_id }}@{{ hostvars[h].ansible_host }}:9093{% if not loop.last %},{% endif %}{% endfor %}


listeners=PLAINTEXT://0.0.0.0:9092,CONTROLLER://0.0.0.0:9093
advertised.listeners=PLAINTEXT://{{ ansible_host }}:9092
controller.listener.names=CONTROLLER
inter.broker.listener.name=PLAINTEXT

log.dirs=/var/lib/kafka
num.partitions=3
default.replication.factor=3
min.insync.replicas=2
offsets.topic.replication.factor=3
04c — Service

templates/kafka.service.j2 + handler

[Unit]
Description=Apache Kafka (KRaft)
After=network.target

[Service]
Type=simple
User=kafka
ExecStart=/opt/kafka/bin/kafka-server-start.sh /opt/kafka/config/kraft/server.properties
ExecStop=/opt/kafka/bin/kafka-server-stop.sh
Restart=on-failure
LimitNOFILE=100000

[Install]
WantedBy=multi-user.target

roles/kafka/handlers/main.yml:

---
- name: restart kafka
  ansible.builtin.systemd: { name: kafka, state: restarted, daemon_reload: true }
  become: true

07 — Health Checks

Verify the cluster

# SSH to broker 1
ssh -i ~/.ssh/kafka-key.pem ec2-user@<broker1-ip>

# Is the service up?
sudo systemctl status kafka --no-pager

# Quorum status (KRaft)
/opt/kafka/bin/kafka-metadata-quorum.sh \
  --bootstrap-server localhost:9092 describe --status

# Create a test topic with 3 replicas
/opt/kafka/bin/kafka-topics.sh --bootstrap-server localhost:9092 \
  --create --topic demo-topic --partitions 3 --replication-factor 3

# Confirm it exists and is fully replicated
/opt/kafka/bin/kafka-topics.sh --bootstrap-server localhost:9092 \
  --describe --topic demo-topic
Healthy signs: systemctl shows active (running), the quorum lists 3 voters with a stable leader, and --describe shows ISR (in-sync replicas) = 3 for every partition.
08 — Python Client

Round-trip test

PLAINTEXT on port 9092 (this demo). Point BROKERS at any broker's reachable IP.

#!/usr/bin/env python3
import json, time
from kafka import KafkaProducer

BROKERS = ["<broker1-ip>:9092", "<broker2-ip>:9092", "<broker3-ip>:9092"]
producer = KafkaProducer(
    bootstrap_servers=BROKERS,
    value_serializer=lambda v: json.dumps(v).encode(),
    acks="all", retries=3,
)
for i in range(10):
    r = producer.send("demo-topic", {"id": i, "ts": time.time()}).get(timeout=10)
    print(f"sent id={i} -> p{r.partition} offset {r.offset}")
producer.flush(); producer.close()
print("done.")
#!/usr/bin/env python3
import json
from kafka import KafkaConsumer

BROKERS = ["<broker1-ip>:9092", "<broker2-ip>:9092", "<broker3-ip>:9092"]
consumer = KafkaConsumer(
    "demo-topic",
    bootstrap_servers=BROKERS,
    auto_offset_reset="earliest",
    group_id="demo-group",
    value_deserializer=lambda b: json.loads(b.decode()),
    consumer_timeout_ms=15000,
)
print("listening...")
for m in consumer:
    print(f"recv p{m.partition} offset {m.offset}: {m.value}")
consumer.close()

Run consumer first, then producer in a second terminal:

python3 consumer.py   # terminal 1
python3 producer.py   # terminal 2
