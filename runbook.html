00 — Background

What we're building, in plain terms

Apache Kafka is a system for moving streams of messages between programs reliably and fast — think of it as a high-throughput post office where "producers" drop messages into named mailboxes ("topics") and "consumers" pick them up. A cluster is several Kafka servers ("brokers") working together so that if one dies, your data and traffic keep flowing.

We use two tools to build it. CloudFormation is AWS's "infrastructure as code" — you write a file describing the servers you want and AWS creates them. Ansible is a configuration tool — once the servers exist, Ansible logs into each one over SSH and installs and configures the software. CloudFormation makes the boxes; Ansible turns the boxes into Kafka brokers.

Modern note: Since Kafka 4.0 (March 2025), ZooKeeper is gone. Kafka now manages its own coordination with KRaft, so there's one less moving part. This runbook uses KRaft throughout.
01 — Two Modes

Two ways to install, one shared role

MODE A

Single Kafka server from one Ansible command

An Ansible playbook spins up one EC2 instance and installs a single-node Kafka broker on it. Fastest way to get a working Kafka for development or a demo. One command, one box.

MODE B

CloudFormation cluster + Ansible controller

CloudFormation provisions a controller EC2 (the Ansible "command" box) plus a cluster of 3 base EC2 brokers. The controller then runs Ansible to install Kafka on all three. This is the production-grade pattern.

03 — Mode A

Single server from one Ansible command

One playbook does both jobs: create the EC2 instance, then (using add_host to register it on the fly) install Kafka. Save as single-server.yml.

---
- name: MODE A — provision one EC2 and install Kafka
  hosts: localhost
  connection: local
  gather_facts: false
  vars:
    region: us-east-1
    keypair: kafka-key
  tasks:
    - name: Look up latest Amazon Linux 2023 AMI
      amazon.aws.ec2_ami_info:
        owners: amazon
        filters:
          name: "al2023-ami-*-x86_64"
          state: available
        region: "{{ region }}"
      register: amis

    - name: Launch single Kafka EC2
      amazon.aws.ec2_instance:
        name: kafka-single
        region: "{{ region }}"
        key_name: "{{ keypair }}"
        instance_type: t3.medium
        image_id: "{{ (amis.images | sort(attribute='creation_date') | last).image_id }}"
        security_group: kafka-sg
        network: { assign_public_ip: true }
        tags: { role: kafka, broker_id: "1" }
        wait: true
      register: ec2

    - name: Register the new host for the next play
      ansible.builtin.add_host:
        name: "{{ ec2.instances[0].public_ip_address }}"
        groups: kafka
        broker_id: "1"
        ansible_user: ec2-user
        ansible_ssh_private_key_file: ~/.ssh/kafka-key.pem
        ansible_ssh_common_args: '-o StrictHostKeyChecking=no'

    - name: Wait for SSH
      ansible.builtin.wait_for:
        host: "{{ ec2.instances[0].public_ip_address }}"
        port: 22
        timeout: 180

- name: Install Kafka on the new instance
  hosts: kafka
  become: true
  vars: { single_node: true }
  roles:
    - kafka

Run it — one command, fully working broker:

ansible-playbook single-server.yml
04 — Mode B

CloudFormation cluster + Ansible controller

CloudFormation builds a controller box and three broker boxes. The controller's UserData installs Ansible and runs the playbook automatically on first boot.

04a — CloudFormation

cluster.yaml

AWSTemplateFormatVersion: '2010-09-09'
Description: Kafka controller + 3 base EC2 brokers (Ansible installs Kafka)

Parameters:
  KeyName: { Type: AWS::EC2::KeyPair::KeyName }
  LatestAmi:
    Type: AWS::SSM::Parameter::Value<AWS::EC2::Image::Id>
    Default: /aws/service/ami-amazon-linux-latest/al2023-ami-kernel-default-x86_64

Resources:
  KafkaSG:
    Type: AWS::EC2::SecurityGroup
    Properties:
      GroupDescription: Kafka + SSH
      SecurityGroupIngress:
        - { IpProtocol: tcp, FromPort: 22,   ToPort: 22,   CidrIp: 0.0.0.0/0 }
        - { IpProtocol: tcp, FromPort: 9092, ToPort: 9093, CidrIp: 10.0.0.0/8 }

  # Role lets the controller read EC2 (for dynamic inventory)
  ControllerRole:
    Type: AWS::IAM::Role
    Properties:
      AssumeRolePolicyDocument:
        Statement: [{ Effect: Allow, Principal: { Service: ec2.amazonaws.com }, Action: sts:AssumeRole }]
      ManagedPolicyArns: [ arn:aws:iam::aws:policy/AmazonEC2ReadOnlyAccess ]
  ControllerProfile:
    Type: AWS::IAM::InstanceProfile
    Properties: { Roles: [ !Ref ControllerRole ] }

  Broker1:
    Type: AWS::EC2::Instance
    Properties:
      ImageId: !Ref LatestAmi
      InstanceType: t3.medium
      KeyName: !Ref KeyName
      SecurityGroupIds: [ !Ref KafkaSG ]
      Tags: [ { Key: role, Value: kafka }, { Key: broker_id, Value: '1' } ]
  Broker2:
    Type: AWS::EC2::Instance
    Properties:
      ImageId: !Ref LatestAmi
      InstanceType: t3.medium
      KeyName: !Ref KeyName
      SecurityGroupIds: [ !Ref KafkaSG ]
      Tags: [ { Key: role, Value: kafka }, { Key: broker_id, Value: '2' } ]
  Broker3:
    Type: AWS::EC2::Instance
    Properties:
      ImageId: !Ref LatestAmi
      InstanceType: t3.medium
      KeyName: !Ref KeyName
      SecurityGroupIds: [ !Ref KafkaSG ]
      Tags: [ { Key: role, Value: kafka }, { Key: broker_id, Value: '3' } ]

  Controller:
    Type: AWS::EC2::Instance
    Properties:
      ImageId: !Ref LatestAmi
      InstanceType: t3.small
      KeyName: !Ref KeyName
      IamInstanceProfile: !Ref ControllerProfile
      SecurityGroupIds: [ !Ref KafkaSG ]
      Tags: [ { Key: role, Value: ansible-controller } ]
      UserData:
        Fn::Base64: !Sub |
          #!/bin/bash
          dnf install -y python3-pip git
          pip3 install ansible boto3 botocore
          ansible-galaxy collection install amazon.aws
          git clone https://example.com/your/kafka-ansible.git /opt/kafka-ansible
          cd /opt/kafka-ansible
          # wait for brokers to be SSH-ready, then configure them
          ansible-playbook -i inventory/aws_ec2.yml site.yml

Outputs:
  ControllerIp: { Value: !GetAtt Controller.PublicIp }
  Broker1Ip:    { Value: !GetAtt Broker1.PublicIp }

Deploy the stack:

aws cloudformation create-stack --stack-name kafka-cluster \
  --template-body file://cluster.yaml \
  --parameters ParameterKey=KeyName,ParameterValue=kafka-key \
  --capabilities CAPABILITY_IAM
04b — Controller

Dynamic inventory + site playbook

inventory/aws_ec2.yml — finds brokers by tag, no hardcoded IPs (Ansible docs):

plugin: amazon.aws.aws_ec2
regions: [ us-east-1 ]
filters:
  tag:role: kafka
  instance-state-name: running
keyed_groups:
  - key: tags.role
    separator: ""
hostnames: [ private-ip-address ]   # controller talks over private network
compose:
  ansible_host: private_ip_address
  broker_id: tags.broker_id

site.yml:

---
- name: Configure Kafka brokers
  hosts: kafka
  become: true
  roles:
    - kafka
04c — Kafka Role (shared)

Best-practice install role

The same role powers both modes. Layout under roles/kafka/:

roles/kafka/
├── tasks/main.yml   # install + configure + start
├── templates/server.properties.j2
├── templates/kafka.service.j2
├── handlers/main.yml  # restart on config change
└── defaults/main.yml  # kafka_version etc.

tasks/main.yml (KRaft, idempotent):

---
- name: Install Java 17
  ansible.builtin.dnf: { name: java-17-amazon-corretto-headless, state: present }

- name: Create kafka user
  ansible.builtin.user: { name: kafka, system: true, shell: /sbin/nologin }

- name: Download & unpack Kafka {{ kafka_version }}
  ansible.builtin.unarchive:
    src: "https://downloads.apache.org/kafka/{{ kafka_version }}/kafka_2.13-{{ kafka_version }}.tgz"
    dest: /opt/
    remote_src: true
    creates: "/opt/kafka_2.13-{{ kafka_version }}"

- name: Symlink /opt/kafka
  ansible.builtin.file:
    src: "/opt/kafka_2.13-{{ kafka_version }}"
    dest: /opt/kafka
    state: link

- name: Data dir
  ansible.builtin.file: { path: /var/lib/kafka, state: directory, owner: kafka, group: kafka }

- name: Render server.properties
  ansible.builtin.template:
    src: server.properties.j2
    dest: /opt/kafka/config/server.properties
    owner: kafka
  notify: restart kafka

- name: Format KRaft storage once
  ansible.builtin.shell: |
    KID=$(/opt/kafka/bin/kafka-storage.sh random-uuid)
    /opt/kafka/bin/kafka-storage.sh format -t $KID \
      -c /opt/kafka/config/server.properties
  args: { creates: /var/lib/kafka/meta.properties }

- name: Install systemd unit
  ansible.builtin.template: { src: kafka.service.j2, dest: /etc/systemd/system/kafka.service }
  notify: restart kafka

- name: Enable + start
  ansible.builtin.systemd: { name: kafka, enabled: true, state: started, daemon_reload: true }

templates/server.properties.j2 — quorum built from inventory:

process.roles=broker,controller
node.id={{ broker_id }}
{% if single_node | default(false) %}
controller.quorum.voters={{ broker_id }}@localhost:9093
default.replication.factor=1
{% else %}
controller.quorum.voters={% for h in groups['kafka'] %}{{ hostvars[h].broker_id }}@{{ hostvars[h].ansible_host }}:9093{% if not loop.last %},{% endif %}{% endfor %}

default.replication.factor=3
min.insync.replicas=2
{% endif %}
listeners=PLAINTEXT://0.0.0.0:9092,CONTROLLER://0.0.0.0:9093
advertised.listeners=PLAINTEXT://{{ ansible_host }}:9092
controller.listener.names=CONTROLLER
inter.broker.listener.name=PLAINTEXT
log.dirs=/var/lib/kafka
num.partitions=3

06 — Health Checks

Confirm it works

# service status
sudo systemctl status kafka --no-pager

# KRaft quorum (expect 3 voters, 1 leader)
/opt/kafka/bin/kafka-metadata-quorum.sh \
  --bootstrap-server localhost:9092 describe --status

# create + describe a replicated topic
/opt/kafka/bin/kafka-topics.sh --bootstrap-server localhost:9092 \
  --create --topic demo --partitions 3 --replication-factor 3
/opt/kafka/bin/kafka-topics.sh --bootstrap-server localhost:9092 \
  --describe --topic demo   # ISR should list 3 replicas
07 — Python Client

Send & receive a message

#!/usr/bin/env python3
import json, time
from kafka import KafkaProducer
p = KafkaProducer(bootstrap_servers=["<broker-ip>:9092"],
                  value_serializer=lambda v: json.dumps(v).encode(),
                  acks="all")
for i in range(10):
    r = p.send("demo", {"id": i, "ts": time.time()}).get(timeout=10)
    print(f"sent {i} -> p{r.partition} offset {r.offset}")
p.flush(); p.close()
#!/usr/bin/env python3
import json
from kafka import KafkaConsumer
c = KafkaConsumer("demo", bootstrap_servers=["<broker-ip>:9092"],
                  auto_offset_reset="earliest", group_id="demo-grp",
                  value_deserializer=lambda b: json.loads(b.decode()),
                  consumer_timeout_ms=15000)
for m in c:
    print(f"recv p{m.partition} offset {m.offset}: {m.value}")
c.close()
Success: producer prints 10 offset confirmations; consumer prints all 10 messages back. Round-trip proven.
